# 快速接入文档

## 1. 接入步骤

1. 引入 starter 依赖。
2. 配置 webhook 地址。
3. 启用需要的告警类型。
4. 启动服务并触发一次异常或慢请求验证。

第一阶段不需要本地安装 Prometheus、SkyWalking、ELK。只要 Spring Boot 应用能启动，并且 webhook 可访问，就可以完成最小验证。

## 2. Maven 依赖示例

```xml
<dependency>
    <groupId>com.pcm.alert</groupId>
    <artifactId>pcm-prometheus-alert-spring-boot-starter</artifactId>
    <version>0.1.0-SNAPSHOT</version>
</dependency>
```

## 3. 配置示例

```yaml
pcm:
  alert:
    enabled: true
    webhook: http://127.0.0.1:8089/mock/webhook
    service-name: order-service
    environment: dev
    exception:
      enabled: true
    request:
      enabled: true
      slow-threshold-ms: 1000
    metric:
      enabled: true
```

## 4. 建议默认行为

1. `enabled=false` 时完全不发送。
2. webhook 缺失时只记录日志，不报错中断业务。
3. 默认只告警高优先级异常。
4. 默认开启基础降噪。

## 5. 验证方式

1. 手动制造一个运行时异常。
2. 发起一个超过阈值的请求。
3. 人工压高 JVM 内存或 CPU 观察告警。

## 6. 接入原则

1. 不要求业务改造主流程。
2. 不要求业务手动调用告警 API。
3. 能自动采集的，不要求人工埋点。

## 7. 本地 mock webhook

如果没有 IM webhook，可先使用 demo 模块提供的接口：

```text
POST /mock/webhook
```

配置方式：

```yaml
pcm:
  alert:
    webhook: http://127.0.0.1:8089/mock/webhook
```
