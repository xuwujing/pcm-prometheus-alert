# 框架说明与启动指南

## 1. 技术选型

MVP 推荐技术栈：

1. Java 8
2. Spring Boot 2.3+
3. Spring Web MVC
4. Spring Boot Actuator
5. Micrometer
6. Jackson
7. Maven

HTTP 客户端建议：

1. 第一阶段使用 Spring `RestTemplate`，便于兼容 Spring Boot 2.x。
2. 后续 Spring Boot 3.x 适配层可支持 `WebClient`。

## 2. 兼容策略

### 2.1 Spring Boot 2.x

主线支持版本。

自动装配文件：

```text
META-INF/spring.factories
```

### 2.2 Spring Boot 3.x

后续支持版本。

自动装配文件：

```text
META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
```

注意事项：

1. `javax.servlet` 需要替换为 `jakarta.servlet`。
2. 自动装配注解和条件装配需要单独验证。

### 2.3 Spring Boot 1.x

长期兼容目标，不进入 MVP。

处理策略：

1. 保持 core 模块 Java 8 兼容。
2. 尽量避免 starter 使用 Spring Boot 2.x 之后才有的强特性。
3. 后续可新增 `spring-boot-1-starter`。

## 3. 工程结构

```text
pcm-prometheus-alert
├── docs
├── pcm-prometheus-alert-core
├── pcm-prometheus-alert-spring-boot-starter
├── pcm-prometheus-alert-demo
├── pcm-prometheus-alert-sql-starter
├── pcm-prometheus-alert-prometheus
└── pom.xml
```

## 4. 本地最小启动方式

MVP 不要求本地安装 Prometheus、SkyWalking、ELK。

最小依赖：

1. JDK 8+
2. Maven 3.6+
3. 一个可接收 POST 请求的 webhook 地址

如果没有真实 IM webhook，demo 模块提供 mock webhook 接口：

```text
POST /mock/webhook
```

## 5. 构建命令

```bash
mvn clean package
```

## 6. 启动 demo

```bash
java -jar pcm-prometheus-alert-demo/target/pcm-prometheus-alert-demo-0.1.0-SNAPSHOT.jar
```

## 7. 本地验证接口

demo 应提供以下接口：

```text
GET /demo/ok
GET /demo/error
GET /demo/slow?millis=1500
POST /mock/webhook
GET /demo/pressure/slow-request
```

## 8. 启动校验点

1. `pcm.alert.enabled=true` 时自动装配生效。
2. `pcm.alert.webhook` 未配置时不发送，只输出日志。
3. `/demo/error` 能产生异常告警。
4. `/demo/slow` 超过阈值后产生慢请求告警。
5. mock webhook 能打印收到的 payload。
