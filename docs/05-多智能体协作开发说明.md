# 多智能体协作开发说明

## 1. 协作目标

将项目拆成独立、可并行、低耦合的开发单元，供后续智能体按文档分批实现。

## 2. 分工建议

1. 架构智能体
   - 维护总体设计
   - 控制技术边界
   - 评审模块拆分

2. 核心开发智能体
   - 实现告警模型
   - 实现 webhook 发送
   - 实现降噪组件

3. Spring 集成智能体
   - 实现自动装配
   - 实现配置绑定
   - 实现异常与请求采集

4. 指标扩展智能体
   - 实现 Micrometer / Actuator 对接
   - 实现 JVM 与运行环境告警

5. SQL 扩展智能体
   - 实现慢 SQL 采集
   - 实现连接池告警

6. 文档智能体
   - 维护接入说明
   - 维护开发记录
   - 维护版本说明

## 3. 任务拆分原则

1. 一个智能体只负责一个模块或一个明确能力。
2. 先写接口和契约，再写实现。
3. 先打通最小闭环，再补充扩展。
4. 文档先行，代码后置。

## 4. 交付要求

每个任务完成后必须同步：

1. 代码变更点
2. 配置变更点
3. 影响范围
4. 验证方式
5. 后续待办

## 5. 禁止事项

1. 不要把 Dubbo 作为基础前提。
2. 不要引入不可替换的内部私有依赖。
3. 不要把所有告警源写进一个巨大类。
4. 不要跳过文档直接堆代码。
