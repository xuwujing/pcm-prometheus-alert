# 开发进度周期说明

## 阶段 0: 设计确认

目标：

1. 确认 MVP 范围
2. 确认模块边界
3. 确认配置项命名
4. 确认 webhook 消息格式

交付物：

1. 总体设计方案
2. 模块拆分说明
3. 配置文档草案

## 阶段 1: 核心骨架

目标：

1. 创建父工程与基础模块
2. 完成 starter 自动装配
3. 完成统一告警模型
4. 完成 webhook 发送器
5. 完成 demo 模块与 mock webhook

交付物：

1. 可编译工程
2. 可接入示例工程
3. 基础配置可生效
4. 本地无需 Prometheus 即可验证异常告警

## 阶段 2: 基础告警能力

目标：

1. Spring MVC 异常告警
2. 慢请求告警
3. JVM 指标告警
4. 告警降噪
5. Spring Boot 2.x 完整验证

交付物：

1. 基础生产可用版本
2. 示例工程验证
3. 最小化回归用例

## 阶段 3: 数据库与 Prometheus 扩展

目标：

1. 慢 SQL 告警
2. 数据源指标告警
3. Prometheus / Actuator 扩展
4. Alertmanager 兼容能力

交付物：

1. 扩展模块
2. 规则说明
3. 配置示例

## 阶段 4: 开源化完善

目标：

1. 文档补齐
2. 示例补齐
3. 配置说明补齐
4. 发布说明补齐

交付物：

1. README
2. 快速接入文档
3. 版本记录

## 版本节奏建议

1. `v0.1.0`：核心骨架
2. `v0.2.0`：异常与请求告警
3. `v0.3.0`：JVM 指标与降噪
4. `v0.4.0`：慢 SQL 与数据源
5. `v1.0.0`：稳定开源版
