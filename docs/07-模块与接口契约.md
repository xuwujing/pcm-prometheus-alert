3. `level`
4. `source`
5. `serviceName`
6. `environment`
7. `host`
8. `traceId`
9. `summary`
10. `detail`
11. `tags`
12. `occurredAt`
13. `stackTrace`
14. `costMs`
15. `requestPath`
16. `requestMethod`

### 3.2 AlertRule

规则对象。

字段建议：

1. `ruleId`
2. `enabled`
3. `type`
4. `level`
5. `threshold`
6. `cooldownSeconds`
7. `windowSize`
8. `recoverable`
9. `include`
10. `exclude`

### 3.3 AlertMessage

字段建议：

1. `title`
2. `content`
3. `level`
4. `type`
5. `webhook`
6. `templateName`
7. `receiver`
8. `occurredAt`

## 5. 推荐核心类

### 5.1 AlertManager

职责：

1. 统一串联规则判断、降噪和发送。
2. 提供 `onEvent(AlertEvent event)` 入口。

建议方法：

```java
public class AlertManager {
    public void onEvent(AlertEvent event);
    public void recover(AlertEvent event);
}
```

### 5.2 WebhookAlertPublisher

职责：

1. 将 `AlertMessage` 序列化为 webhook 请求。
2. 做超时控制和失败日志。

建议方法：

```java
public class WebhookAlertPublisher implements AlertPublisher {
    public void publish(AlertMessage message);
    protected String buildPayload(AlertMessage message);
}
```

### 5.3 AlertProperties

职责：

1. 绑定 `pcm.alert.*` 配置。
2. 提供默认值。

建议字段：

1. `enabled`
2. `webhook`
3. `serviceName`
4. `environment`
5. `slowRequestThresholdMs`
6. `cooldownSeconds`
7. `queueSize`
8. `timeoutMs`

### 5.4 PcmAlertAutoConfiguration

职责：

1. 注册 `AlertProperties`
2. 注册 `AlertManager`
3. 注册 `AlertPublisher`
4. 注册采集器
5. 注册降噪器

### 5.5 MvcExceptionCollector

职责：

1. 通过 `HandlerExceptionResolver` 或 `@ControllerAdvice` 获取异常。
2. 生成异常事件。

### 5.6 SlowRequestCollector

职责：

1. 统计请求耗时。
2. 判断是否超过阈值。

### 5.7 MetricCollector

职责：

1. 读取 JVM / CPU / 线程指标。
2. 定时触发事件。

## 6. 关键方法链路

### 6.1 异常告警链路

1. `MvcExceptionCollector` 捕获异常。
2. 调用 `AlertEventFactory.fromException(...)`。
3. `AlertManager.onEvent(...)`。
4. `AlertRuleEvaluator.shouldAlert(...)`。
5. `AlertDeduplicator.allow(...)`。
6. `AlertMessageRenderer.render(...)`。
7. `AsyncAlertPublisher.submit(...)`。
8. `WebhookAlertPublisher.publish(...)`。

### 6.2 慢请求链路

1. `RequestCollector` 记录开始时间。
2. 请求结束后计算耗时。
3. 超过阈值生成 `SLOW_REQUEST` 事件。
4. 进入统一告警链路。

### 6.3 指标告警链路

1. `MetricCollector` 定时采样。
2. 生成 `JVM_MEMORY`、`CPU_USAGE`、`THREAD_COUNT` 事件。
3. 进入统一告警链路。
