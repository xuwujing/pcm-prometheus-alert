### 下午：请求告警完善（路径排除 + 状态码告警）

22. **AlertEvent** 新增 `statusCode` 字段。
23. **AlertType** 新增 `HTTP_STATUS` 枚举值。
24. **AlertProperties.Request** 新增三项配置：
    - `excludePaths`：Ant 风格路径排除，默认 `[/actuator/**, /health, /favicon.ico]`
    - `statusCodeAlertEnabled`：HTTP 状态码告警开关，默认 true
    - `statusCodeAlertThresholds`：触发告警的状态码列表，默认 `[500, 502, 503, 504]`
25. **SlowRequestFilter** 改造：
    - 新增 `shouldNotFilter()` 实现路径排除
    - 新增 `checkStatusCode()` 在请求完成后检查响应状态码
26. **SpringAlertEventFactory** 新增 `fromHttpStatus()` 方法。
27. **DefaultAlertMessageRenderer** 渲染时输出 `statusCode` 字段。
28. **DemoController** 新增 `/demo/status-500` 测试接口。
29. 编写 starter 模块单元测试：
    - `SpringAlertEventFactoryTest`（6 个用例）
    - `SlowRequestFilterTest`（7 个用例）
30. starter/pom.xml 新增 test 依赖（spring-boot-starter-test、junit-vintage-engine、mockito-core）。
31. 全量 19 个测试通过（core 3 + starter 13 + demo 3）。
